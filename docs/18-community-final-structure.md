# 웨더체크 현장 커뮤니티 최종 구조

## 1. 커뮤니티의 한 문장 정의

웨더체크의 커뮤니티는 **궁금한 지역에 질문하고, 그 지역에 실제로 있는 사람이 답하며, 전국의 최신 현장 날씨를 함께 보는 기능**이다.

단순 게시판이나 자유 커뮤니티로 확장하지 않는다. 모든 글은 현재 날씨 확인이라는 목적을 가져야 한다.

## 2. 메인 탭 이름

기존 `제보`는 질문과 답변까지 포함하기에 의미가 좁다.

메인 내비게이션 이름은 **`현장`**으로 변경한다.

상단 제목:

- 제목: `현장 날씨`
- 설명: `궁금한 곳은 묻고, 가까운 곳은 직접 알려주세요`

## 3. 화면 구조

상단에는 하나의 세그먼트 컨트롤만 둔다.

`내 문의 | 답변하기 | 현장 소식`

각 화면은 한 가지 역할만 수행한다.

### 내 문의

목적: 다른 지역의 지금 날씨를 묻고, 내가 올린 질문과 답변을 확인한다.

구성:

- 큰 질문 입력창
- 장소 검색 결과 확인
- `문의 보내기` 버튼
- 내가 올린 문의 목록
- 상태 표시: `답변 대기`, `답변 2개`, `종료`
- 문의를 누르면 질문과 답변을 한 화면에서 보는 상세 바텀시트
- 내가 쓴 문의 수정 및 삭제

질문 예:

- `광안리 지금 비 와요?`
- `서서울CC 바닥 많이 젖었나요?`
- `설악산 입구에 안개 심한가요?`

### 답변하기

목적: 현재 위치에서 실제로 확인할 수 있는 질문에만 답한다.

구성:

- 기본 목록은 현재 위치에서 약 3km 안의 미답변 또는 진행 중 질문
- 장소, 거리, 작성 시각, 답변 수 표시
- 질문을 누르면 기존 답변과 답변 입력창을 함께 표시
- 현재 위치를 확인하지 못했거나 질문 지역에서 멀면 답변 입력을 막고 이유 표시
- 답변을 남기면 해당 질문 상세와 현장 소식에 즉시 반영
- 답변할 질문이 없으면 깔끔한 빈 화면과 위치 새로고침 버튼 표시

전국의 답변 불가능한 질문은 이 화면에 섞지 않는다.

### 현장 소식

목적: 전국에서 실제로 올라온 최신 날씨 정보를 시간순으로 본다.

포함되는 글:

- 요약 탭에서 남긴 현재 위치 날씨 제보
- 지역 질문에 달린 현장 답변

구성:

- 최신순 단일 피드
- 글 종류를 작은 라벨로만 구분: `현장 제보`, `문의 답변`
- 장소, 작성 시각, 날씨 상태, 본문 표시
- 피드 전체를 가르는 불필요한 카드 색상은 사용하지 않는다
- 내가 쓴 글은 수정 및 삭제
- 다른 사람 글은 신고
- 신고한 글은 `신고 접수됨`으로 표시

`근처/전국` 토글은 기본 화면에서 제거한다. 현장 소식은 전국 최신순으로 단순하게 유지하고, 지도에서 지역별 탐색을 담당한다.

## 4. 글의 관계

서버 데이터는 아래 두 종류만 유지한다.

1. `report_request`: 사용자의 질문
2. `field_report`: 현장 날씨 글

`field_report.requestId`가 있으면 질문에 대한 답변이고, 없으면 독립적인 현재 위치 제보다.

화면에서 질문과 답변을 별도 복제하지 않는다. 같은 서버 데이터를 각 목적에 맞게 보여준다.

## 5. 위치와 개인정보 원칙

- 질문 위치는 질문자가 궁금해하는 장소다.
- 답변 위치는 답변자가 실제로 있는 현재 위치로 검증한다.
- 독립 현장 제보는 작성자의 현재 위치에서만 작성할 수 있다.
- 건물명, 동·호수, 정확한 좌표는 공개하지 않는다.
- 공개 지도에는 약 1.5km 단위로 흐린 위치만 표시한다.
- 답변 가능 여부는 현재 위치와 질문 지역 간 약 3km 기준으로 서버에서 최종 검증한다.
- 지도 마커는 작성자 위치가 아니라 질문 대상 지역 또는 공개용으로 흐린 제보 지역에 표시한다.

## 6. 시간 정책

날씨 정보가 오래 남아 오해를 만들지 않도록 시간 정책을 고정한다.

- 질문 답변 가능 시간: 작성 후 2시간
- 현장 소식 기본 노출: 작성 후 24시간
- 지도 묶음에 포함: 최근 6시간
- 내 문의 보관: 7일
- 종료된 질문에는 새 답변을 받을 수 없다

모든 카드에는 `방금`, `12분 전`, `오늘 14:20`처럼 실제 작성 시간을 표시한다.

## 7. 다른 탭과 연결

- 요약의 `현장에 물어보기`는 `현장 > 내 문의`를 열고 검색 장소를 미리 채운다.
- 요약의 `지금 날씨 남기기`는 현재 위치 제보 작성창을 연다.
- 지도 묶음 마커를 누르면 해당 지역의 현장 소식을 바텀시트로 연다.
- 지도 바텀시트의 질문을 누르면 `현장 > 답변하기` 상세로 연결한다.
- 같은 작성 기능을 화면마다 따로 만들지 않고 하나의 공용 작성창을 사용한다.

## 8. 상태와 문구

질문 상태:

- `답변 대기`
- `답변 1개`
- `종료`

전송 상태:

- `보내는 중`
- `등록됨`
- `전송 실패 · 다시 시도`

신고 상태:

- `신고`
- `신고 접수됨`

`검토중`, `현장 우세`, `질문 지역`처럼 의미를 다시 해석해야 하는 표현은 사용하지 않는다.

## 9. 출시 전 필수 기능

- 질문, 답변, 제보의 서버 저장과 다른 기기 동기화
- 작성자 본인의 수정 및 삭제
- 위치 기반 답변 제한의 서버 검증
- 질문 만료 및 오래된 현장 글 자동 제외
- 신고와 관리자 숨김 처리
- 질문에 답변이 달렸을 때 앱 안 알림 배지
- 빈 화면, 전송 실패, 위치 권한 거부 상태
- 가상 질문과 샘플 제보 완전 제거

푸시 알림은 스토어 출시 전 추가한다. 첫 외부 테스트에서는 앱 안 알림 배지와 주기적 동기화로 검증할 수 있다.

## 10. 구현 순서

1. 탭과 내부 메뉴 명칭을 `현장 / 내 문의 / 답변하기 / 현장 소식`으로 변경
2. 답변하기 목록은 클라이언트에서 3km 이내 질문만 노출하고, 답변 등록 시 서버가 3km를 다시 검증
3. 질문 2시간, 피드 24시간, 지도 6시간 만료 규칙 구현
4. 질문 상세 바텀시트에서 답변 조회와 작성 통합
5. 현장 소식을 단일 최신순 피드로 정리
6. 요약 및 지도에서 같은 작성·상세 화면으로 연결
7. 실제 두 기기 동기화와 위치 경계 테스트
