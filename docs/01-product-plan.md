# 웨더체크 제품 기획서 v0.1

이 문서는 지금까지 만든 목업을 실제 앱으로 만들기 위한 기준 문서입니다.
코딩을 모르는 사람도 "우리가 뭘 만들고 있는지" 확인할 수 있게 적었습니다.

## 한 줄 설명

웨더체크는 여러 예보 서비스와 사람들의 현장 제보를 비교해서
"지금 이 장소에 비가 오는지"를 판정해주는 모바일 앱입니다.

## 사용자 문제

사용자는 외출, 야구장, 콘서트, 등교, 출근처럼 실제 행동을 결정해야 할 때
단순한 예보보다 "지금 거기 진짜 비 와?"라는 답이 필요합니다.

## 핵심 가치

- 예보를 하나만 믿지 않고 여러 서비스를 비교합니다.
- 현장 제보를 함께 보여줘서 실제 상황을 보완합니다.
- 어려운 기상 정보보다 바로 행동에 도움 되는 답을 먼저 보여줍니다.

## 주요 화면

### 1. 판정 탭

첫 화면입니다.

사용자가 앱을 열면 현재 위치 기준으로 "현재는 비 안 오는 쪽이 우세해요" 같은 답을 바로 봅니다.
상단 검색창에는 "잠실운동장 지금 비 와?"처럼 질문을 입력할 수 있습니다.

판정 카드에는 다음 정보가 들어갑니다.

- 판정 문장
- 큰 날씨 아이콘
- 현재 기온
- 판정 단계: 확실, 중간, 애매, 위험
- 현장 제보 수
- 예보 서비스들의 현재 날씨와 기온
- 사용자가 지금 날씨를 한 줄로 남기는 입력 영역

### 2. 지도 탭

주변 사람들이 올린 현장 날씨 글을 지도 위에서 봅니다.

초기 버전에서는 실제 지도 API를 바로 붙이기 전에,
지도 모양 UI와 제보 핀으로 먼저 구현합니다.
이후 Kakao Map, Naver Map, Mapbox 중 하나를 선택합니다.

### 3. 제보 탭

사용자들이 직접 질문하고 답하는 공간입니다.

초기 버전에서는 특정 사용자에게 알림을 보내는 방식보다
"제보 요청 리스트"를 만들고, 사용자가 원하는 요청을 선택해서 답하는 방식으로 갑니다.
이 방식이 구현 난이도가 낮고, 개인정보 부담도 적습니다.

### 4. 비교 탭

기상청, Yr.no, FMI ECMWF 예보를 시간별/날짜별로 비교합니다.

목표는 네이버 날씨의 예보비교처럼 보기 쉽게 만들되,
텍스트만 나열하지 않고 날씨 아이콘과 기온을 같이 보여주는 것입니다.

## MVP에 넣을 기능

MVP는 첫 출시 전 최소 제품입니다.
처음부터 모든 기능을 넣으려 하면 느려지므로, 아래 기능까지만 1차 목표로 둡니다.

- 현재 위치 기반 판정 화면
- 장소 검색 질문 입력 UI
- 샘플 데이터 기반 판정 결과 표시
- 날씨별 판정 카드 디자인
- 현장 날씨 한 줄 남기기
- 제보 요청 리스트와 답변 UI
- 시간별/날짜별 예보 비교 UI
- 지도 탭의 제보 핀 UI

## MVP 이후 기능

아래 기능은 바로 만들지 않고, 앱의 기본 흐름이 안정된 뒤에 붙입니다.

- 실제 기상청 API 연결
- Yr.no API 연결
- FMI ECMWF 데이터 연결
- Windy 데이터 연동 가능성 검토
- 사용자 로그인
- 현장 제보 서버 저장
- 위치 기반 푸시 알림
- 서비스별 과거 적중률 계산
- 지도 API 정식 연결
- 앱스토어 심사용 개인정보 처리방침 정리

## 데이터 개념

아직 데이터베이스를 만들지는 않았지만, 나중에는 이런 정보가 필요합니다.

### WeatherJudgment

한 장소와 시간에 대한 최종 판정입니다.

- 장소
- 시간
- 판정 문장
- 날씨 상태
- 기온
- 판정 단계
- 근거 요약

### ForecastSource

예보 서비스별 데이터입니다.

- 서비스명: 기상청, Yr.no, FMI ECMWF
- 날씨 아이콘
- 기온
- 강수 가능성
- 시간별 예보
- 날짜별 예보

### LocalReport

사용자가 남긴 현장 제보입니다.

- 작성 위치
- 작성 시간
- 날씨 상태
- 한 줄 내용
- 사진 여부

### ReportRequest

다른 사용자에게 물어보는 제보 요청입니다.

- 요청 장소
- 요청 질문
- 요청 시간
- 답변 수
- 상태: 대기, 답변 있음, 종료

## 지금의 중요한 결정

웹 목업은 그대로 참고하되, 실제 개발은 React Native + Expo로 시작합니다.
이유는 Android와 iOS 앱을 모두 출시해야 하기 때문입니다.
