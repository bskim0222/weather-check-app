# 판정 멘트 가이드

날씨체크의 핵심 문장은 사용자가 바로 행동을 정할 수 있게 도와야 한다. 다만 예보 API만으로는 실제 현장 상태를 100% 단정할 수 없으므로, 문구는 "확정"보다 "근거 있는 판단"으로 표현한다.

## 기본 원칙

- 현재 실제 날씨를 직접 관측한 것처럼 단정하지 않는다.
- 서비스별 예보가 몇 곳이나 같은 방향인지 함께 말한다.
- 위험 날씨는 확률이 낮아도 더 조심스럽게 표현한다.
- 현장 제보는 예보와 분리해서 보여주고, 제보가 쌓였을 때만 판정 근거로 강하게 반영한다.
- 사용자가 이해하기 어려운 내부 용어를 쓰지 않는다. 예: 컨센서스, 신뢰도, 가중치.

## 판정 단계

| 단계 | 의미 | 사용자 문구 방향 |
| --- | --- | --- |
| 확실 | 3개 서비스가 거의 같은 방향 | "비 걱정은 낮은 편이에요", "비 가능성이 더 높아요" |
| 우세 | 3개 중 2개 이상이 같은 방향 | "쪽 신호가 더 많아요", "가능성이 더 높아요" |
| 엇갈림 | 서비스별 예보가 갈림 | "신호가 엇갈려요", "현장 제보 확인이 필요해요" |
| 주의 | 천둥, 눈, 안개처럼 행동 영향이 큰 상황 | "확인해야 해요", "주의가 필요해요" |

## 날씨별 대표 문장

### 비

- 우세 이상: "비 가능성이 더 높아요"
- 엇갈림: "비 신호가 엇갈려요"
- 보조 설명: "3곳 중 2곳이 비 쪽이라 이렇게 판단했어요."

### 맑음

- 우세 이상: "비 걱정은 낮은 편이에요"
- 엇갈림: "비 가능성은 낮아 보여요"
- 보조 설명: "대부분 서비스가 강수 없음 쪽으로 보고 있어요."

### 흐림

- 기본: "비보다는 흐림 쪽 신호예요"
- 보조 설명: "흐림은 보여도 강수 신호는 약한 편이에요."

### 천둥번개

- 기본: "소나기·천둥 가능성을 확인해야 해요"
- 보조 설명: "천둥이나 소나기 신호는 짧게 바뀔 수 있어 현장 확인이 중요해요."

### 눈

- 기본: "눈 또는 진눈깨비 신호가 있어요"
- 보조 설명: "눈 신호는 이동에 영향이 커서 보수적으로 안내해요."

### 안개

- 기본: "비보다 시야 확인이 중요해요"
- 보조 설명: "강수보다 안개와 낮은 구름 신호를 우선 확인했어요."

## 화면별 역할

- 판정카드: 사용자가 가장 먼저 볼 직접 답변.
- 이렇게 판단했어요: 서비스별 현재 날씨와 기온을 보여주는 근거.
- 다음 6시간 날씨: 지금 판정 이후 흐름.
- 현재 위치 제보: 실제 현장 정보 보강.
- 비교탭: 서비스별 시간대/날짜별 차이를 표로 확인하는 곳.

## 피해야 할 문장

- "현재 비가 옵니다"처럼 현장 관측처럼 들리는 표현.
- "정확도 90%"처럼 검증되지 않은 수치.
- "기상청이 틀렸어요"처럼 특정 서비스를 직접 비난하는 표현.
- "확실히 안 와요"처럼 사용자의 행동을 위험하게 만들 수 있는 단정.

## 좋은 문장 구조

1. 결론: "비 가능성이 더 높아요"
2. 근거: "3곳 중 2곳이 비 쪽이에요"
3. 보완: "근처 제보가 들어오면 함께 반영해요"

이 구조를 유지하면 사용자는 판정을 빠르게 이해하면서도, 앱이 왜 그렇게 말하는지 납득할 수 있다.
